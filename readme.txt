Add validated custom JSON-LD to individual posts, pages, products, and public custom post types.

== Description ==

SafeSchema adds a lightweight meta box below the WordPress editor. Paste raw JSON-LD or a complete JSON-LD script block, validate it, and choose one output mode:

* Disabled: no custom schema is printed.
* Add: custom schema is printed alongside existing schema.
* Replace supported SEO schema: schema generated by supported SEO plugins is disabled for that singular URL, then SafeSchema prints the custom schema.

Supported SEO integrations in version 1.0 beta:

* Rank Math
* Yoast SEO
* All in One SEO
* SEOPress, beta compatibility

SafeSchema does not disable SEO titles, meta descriptions, canonical tags, Open Graph tags, sitemaps, robots directives, or other non-schema SEO features.

Important: Replace mode only targets supported SEO plugin schema. Themes, WooCommerce, blocks, tag managers, and unrelated plugins can still output structured data. Always inspect the final page and test the public URL.

== Installation ==

1. Upload and activate SafeSchema.
2. Edit a post, page, product, or public custom post type.
3. Scroll to the SafeSchema meta box.
4. Paste JSON-LD and click Validate.
5. Choose Add or Replace supported SEO schema.
6. Update or publish the content.
7. Test the public URL with Google Rich Results Test and Schema.org Validator.

== Frequently Asked Questions ==

= Does SafeSchema validate Google rich result eligibility? =

No. It validates JSON syntax and basic JSON-LD structure. Google eligibility depends on the schema type, required properties, page content, and current search documentation.

= Does Replace mode remove every schema block from the page? =

No. It suppresses schema from supported SEO plugins. Other components can still output JSON-LD or microdata.

= What happens when invalid JSON is saved? =

WordPress saves the post content, but SafeSchema keeps the previously valid schema and mode. It shows an admin notice explaining the validation error.

= Does it work with Gutenberg? =

Yes. It uses a Gutenberg-compatible meta box below the editor.
